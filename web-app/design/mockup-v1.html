SQLite vs PG

今天
我 我
为什么我看 benchmark SQLite 在小事务上比 Postgres 快这么多？
C Claude

核心是它跑在你进程里，没有跨进程通信、没有连接握手、没有线性可写日志同步给独立 daemon 的开销。

SQLite 一次小写就是几次 syscall，而 PG 即便最优化也要：客户端 → 网络 → backend 进程 → WAL → fsync → 返回。

  • 语音输入是第一组，最显眼。toggle 默认开——多数人想要顺口
  • iOS 风分组卡片，每组带顶部标题，圆角 + 软分隔

说明 / 待定

  • 这是静态稿——不能点不能滚，只用来对方向。React 落地按这版做
  • 头像方案：列表用 40 圆角方块，对话内用 18 圆点 + 字母。如果你喜欢「全程无头像」（更 Claude.ai）也能去掉
  • 建议输入（新对话首屏的 3 个 chip）目前是写死的；可以后续加规则或让 Claude 根据时间/最近聊过的话题动态出
  • 语音 7-12：录音、转写中、转写完、整理后、设置——5 个状态都画了。麦克风按钮位置和波形动画方向由你定，不喜欢可以小调
  • 暗色模式晚一步，先跑通亮色
  • 没设计的：长按消息（复制/重发）、对话重命名、删除对话、长按麦克风「按住录音/松开发送」（vs 现在的「点一下/再点一下」）——这些都是后续迭代
